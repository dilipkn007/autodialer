# FOLK Auto Dialer

> Follow-up Management & Smart Calling System

## Overview

FOLK Auto Dialer is a Flutter mobile application designed to streamline the process of calling a large list of contacts and collecting structured follow-up information. The system operates with two user personas:

1. **Admin (Folk Guide)** — The central coordinator who manages enablers, creates events with custom survey forms, uploads/assigns contacts, generates access tokens, and views analytics.
2. **Enabler** — A field caller who receives an assigned list of contacts, initiates calling sessions via an auto-dialer, and records responses through dynamic forms after each call.

## Problem Statement

Currently, follow-up data is managed via large Excel spreadsheets containing 1800+ contact records with 40+ columns of member information. The manual process involves:

- Distributing contact lists to enablers
- Enablers manually calling each contact one-by-one
- Recording responses in spreadsheets
- Manually consolidating responses back to the admin

## Solution

The app automates the entire calling workflow:

1. **Admin creates/imports contacts** into a PostgreSQL database via Supabase, with CSV bulk upload support
2. **Admin creates events** with custom calling survey forms (dynamic questions with dropdown, radio, checkbox, text, and date types)
3. **Admin assigns contacts** to specific enablers for a given event
4. **Enablers log in** using an access token (no OTP required) and see their assigned contact list
5. **Enablers start the auto-dialer** which initiates an automated calling session:
   - The app calls the first contact via the phone dialer
   - After the call ends, a dynamic response form appears
   - The enabler fills in call outcome, follow-up status, and custom survey answers
   - A configurable countdown timer (5–60 seconds) begins
   - When the timer elapses (or the enabler taps next), the app automatically calls the next contact
6. **All responses are synced** to the database in real-time
7. **Admin views dashboards** with analytics, completion rates, and enabler performance

## Architecture

### Auth Flow

Two authentication methods:

- **Admin Login** — Phone number + OTP verification via Supabase Auth. New users register with name/email (role: ENABLER). Admin accounts are provisioned via the `contact` table with `role = 'ADMIN'`.
- **Enabler Login** — Access token generated by admins. The `login-with-token` Supabase Edge Function validates the token, creates/looks up the auth user, and returns one-time credentials for sign-in.

Admins can switch between admin and enabler mode per session via the welcome screen.

## Database Schema (Supabase / PostgreSQL)

### Core Tables

#### `contact` — Unified contacts/users table
Stores both member profiles and app users (admins/enablers), distinguished by `role`.

Key columns: `id` (uuid PK), `name`, `mobile` (unique), `email`, `whatsapp`, `role` (ADMIN|ENABLER), along with 40+ demographic and engagement fields.

#### `event` — Calling campaigns
`id`, `name`, `description`, `event_date`, `event_time`, `audience_filter`, `status`, `gap_duration` (seconds between auto-dialer calls), `created_by`.

#### `assignment` — Contacts assigned to enablers for events
`id`, `contact_id`, `enabler_id`, `event_id`, `assigned_by`, `status`, `sort_order`.

#### `call_log` — Record of each call
`id`, `assignment_id`, `contact_id`, `enabler_id`, `event_id`, `call_outcome`, `follow_up_status`, `follow_up_notes`, `next_call_date`, `call_duration`.

#### `survey_question` — Dynamic form questions per event
`id`, `event_id`, `question_title`, `question_type` (DROPDOWN|TEXT|DATE|MULTI_SELECT|RADIO), `options`, `sort_order`, `is_required`.

#### `survey_response` — Answers to survey questions
`id`, `call_log_id`, `question_id`, `answer`.

#### `access_token` — Login tokens for enablers
`id`, `contact_id`, `token`, `mobile_number`, `expires_at`, `revoked`, `login_count`, `last_login_at`.

#### `ai_chat_session` / `ai_chat_message` — AI Assistant chat history
Stores Gemini chat sessions and messages with full parts JSON for history replay.

#### `app_config` — Key-value configuration
Stores `gemini_api_key`, `gemini_input_tokens`, `gemini_output_tokens`, `gemini_budget`.

### RPC Functions
- `migrate_contact_identity(p_old_id, p_new_id, p_mobile, p_name, p_role)` — Migrates pre-created "dummy" profiles to new auth user IDs
- `run_dynamic_sql(query_string)` — Read-only SQL execution for AI Assistant
- `execute_mutation(sql_string)` — Write SQL execution for AI Assistant

## Key Features

### Admin Features

- **Dashboard** — Stats cards, call outcome pie chart, campaign progress, recent activity feed
- **Contact Assignment** — Browse/search/filter 1800+ contacts, bulk select and assign to enabler+event, CSV export/import
- **Enablers Management** — Paginated list with search, completion stats per enabler, weekly trends chart, activation toggle
- **Events** — CRUD with audience filters, dynamic survey builder (drag-and-drop reorder, 5 question types, presets)
- **Event Analytics** — Per-event detailed analytics
- **AI Assistant** — Gemini 2.5 Flash chat that can query and mutate the database using natural language. Includes speech-to-text, session management, token budget tracking, and ISKCON-themed persona
- **Access Management** — Generate/revoke UUID-based access tokens, browse contacts, share tokens via WhatsApp

### Enabler Features

- **Assigned Contacts** — List of contacts assigned by admin, filterable by event and call status, searchable
- **Calling Dashboard** — Single contact view with call/message/info actions, outcome dropdown, follow-up status, notes field, next call date, dynamic survey questions
- **Auto Dialer** — Automated calling queue with gap timer, auto-calls next contact, pause/resume, dynamic survey form, auto-saves responses

## Getting Started

### Prerequisites
- Flutter SDK >= 3.0.0
- A Supabase project (URL and anon key configured in `lib/main.dart`)
- The `login-with-token` Edge Function deployed
- Database migrations applied

### Setup

```bash
# Clone the repository
git clone <repo-url>
cd autodialer

# Install dependencies
flutter pub get

# Configure Supabase
supabase link --project-ref <your-project-ref>

# Deploy edge function
supabase functions deploy login-with-token

# Apply migrations
supabase db push

# Run the app
flutter run
```

### Configuration
- **Supabase credentials** — Set the URL and anon key in `lib/main.dart`
- **Gemini API key** — Insert a row into `app_config` with `key = 'gemini_api_key'`

### Build
```bash
flutter build apk           # Android
flutter build ios            # iOS
flutter build web            # Web
```
